# Comparison: L2 vs Cosine vs Dot product (Inner product / MIP)

| **Aspect**                                            | **L2 Distance (Euclidean)**                        | **Cosine Similarity**                                 | **Inner Product (Dot Product / MIP)**                                      |     |     |     |     |     |     |     |     |     |     |     |     |
| ----------------------------------------------------- | -------------------------------------------------- | ----------------------------------------------------- | -------------------------------------------------------------------------- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| **What it fundamentally measures**                    | Absolute **geometric distance** between two points | **Angle between vectors** (direction similarity)      | **Projection strength** (alignment × magnitude)                            |     |     |     |     |     |     |     |     |     |     |     |     |
| Behavior after normalization                          | Becomes equivalent to cosine ranking               | No change                                             | Becomes identical to cosine                                                |     |     |     |     |     |     |     |     |
| **In RAG System (normalized vectors)**                | Shortest between two vectors                       | Encoded similarity                                    | Same as Cosine. Encoded similarity                                         |     |     |     |     |     |     |     |     |     |     |     |     |
| **Speed (Hardware)**                                  | Extremely fast on CPUs                             | Extremely fast on GPUs/TPUs                           | Fast because its same as cosine.                                           |     |     |     |     |     |     |     |     |     |     |     |     |
| **Optimization goal (how FAISS uses it)**             | **Minimize** distance                              | **Maximize** similarity                               | **Maximize** similarity                                                    |     |     |     |     |     |     |     |     |     |     |     |     |
| **Range (possible values)**                           | ([0, +inf))                                        | ([-1, 1])                                             | ((-inf, +inf))                                                             |     |     |     |     |     |     |     |     |     |     |     |     |
| **Meaning of extreme values**                         | 0 → identical vectors                              | 1 → same direction<br>0 → orthogonal<br>-1 → opposite | Large positive → strong alignment<br>0 → orthogonal<br>Negative → opposite |     |     |     |     |     |     |     |     |     |     |     |     |
| **Monotonic behavior (ranking intuition)**            | Smaller = more similar                             | Larger = more similar                                 | Larger = more similar                                                      |     |     |     |     |     |     |     |     |     |     |     |     |
| **Sensitivity to magnitude (vector length)**          | ✅ Strongly affected                               | ❌ Completely ignores magnitude                       | ✅ Strongly affected                                                       |     |     |     |     |     |     |     |     |     |     |     |     |
| **Sensitivity to direction (angle)**                  | ⚠️ Indirect (via coordinates)                      | ✅ Purely directional                                 | ✅ Direction + magnitude combined                                          |     |     |     |     |     |     |     |     |     |     |     |     |
| **Effect of scaling a vector (e.g., multiply by 10)** | Distance increases                                 | No change                                             | Score increases                                                            |     |     |     |     |     |     |     |     |     |     |     |     |
| **Normalization requirement**                         | ❌ Not required                                    | ✅ Required for correctness                           | ⚠️ Optional (depends on use case)                                          |     |     |     |     |     |     |     |     |     |     |     |     |

| **Metric properties (distance axioms)** | ✅ True metric (triangle inequality, symmetry) | ❌ Not a proper metric | ❌ Not a metric | | | | | | | | | | | | |
| **FAISS support** | Native (`IndexFlatL2`) | Via normalization + IP | Native (`IndexFlatIP`) | | | | | | | | | | | | |
| **Score interpretability** | Real distance (physical meaning) | Pure similarity score (bounded) | Raw similarity signal (unbounded) | | | | | | | | | | | | |
| **Typical output distribution (embeddings)** | Wide positive range | Mostly 0 → 1 (rarely negative) | Depends on magnitude distribution | | | | | | | | | | | | |
| **Common use cases** | Vision, geometry, clustering | NLP, semantic search, RAG | Recommendation systems, retrieval | | | | | | | | | | | | |
| **Risk / pitfalls** | Sensitive to scale differences | Negative values often misunderstood | Magnitude bias can dominate | | | | | | | | | | | | |
| **When it breaks down** | When vectors differ in scale | When embeddings are not normalized | When vectors are not normalized | | | | | | | | | | | | |
